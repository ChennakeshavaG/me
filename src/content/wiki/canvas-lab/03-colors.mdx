## How Computers Represent Color

<p>Before we paint anything, let's understand how color works on screens. Your monitor has millions of tiny lights, each made of <strong>three sub-pixels</strong>: one red, one green, and one blue. By mixing these three colors of light at different intensities, your screen can create virtually any color you can see.</p>

<p>This is called the <strong>RGB color model</strong>. Each channel (Red, Green, Blue) ranges from 0 (off) to 255 (full brightness). That gives us 256 x 256 x 256 = over 16 million possible colors.</p>

<Concept>
  <p><strong>Think of it like stage lighting.</strong> Imagine three spotlights in a dark theater: one red, one green, one blue. Point them all at the same spot on the wall:</p>
  <pre>{`
  Red only       Green only     Blue only      Red + Green    All three
  +------+       +------+       +------+       +------+       +------+
  | R    |       |    G |       |      |       | R  G |       | R  G |
  |  **  |       |  **  |       |  **  |       |  **  |       |  **  |
  | RED  |       | GRN  |       |   B  |       | YLW  |       |   B  |
  +------+       +------+       +------+       +------+       +------+
  (255,0,0)      (0,255,0)      (0,0,255)     (255,255,0)   (255,255,255)
                                                 = Yellow       = White!`}</pre>
  <p>Key insight: mixing <em>light</em> is <strong>additive</strong>. Adding more light makes things brighter. This is the opposite of mixing paint, where adding more paint makes things darker.</p>
</Concept>

<p>The demos below use <code>beginPath()</code> and <code>arc()</code> from Lesson 2 to draw circles. Here, we focus on <em>color</em> rather than shape.</p>

export const rgbMixingCode = `// Draw three overlapping circles to show additive color mixing
// Just like three colored spotlights on a stage

var centerX = 250;  // center of the canvas
var centerY = 160;  // vertical center for the circles
var radius = 90;    // how big each circle is
var spread = 60;    // how far apart the centers are

// Enable "lighter" blending (a composite mode covered later in this lesson)
// that simulates real light mixing
ctx.globalCompositeOperation = 'lighter';
ctx.globalAlpha = 0.6;

// Red spotlight (top-left position)
ctx.fillStyle = 'rgb(255, 0, 0)';
ctx.beginPath();
ctx.arc(centerX - spread, centerY - 30, radius, 0, Math.PI * 2);
ctx.fill();

// Green spotlight (top-right position)
ctx.fillStyle = 'rgb(0, 255, 0)';
ctx.beginPath();
ctx.arc(centerX + spread, centerY - 30, radius, 0, Math.PI * 2);
ctx.fill();

// Blue spotlight (bottom-center position)
ctx.fillStyle = 'rgb(0, 0, 255)';
ctx.beginPath();
ctx.arc(centerX, centerY + 50, radius, 0, Math.PI * 2);
ctx.fill();

// Reset compositing mode and alpha
ctx.globalAlpha = 1;
ctx.globalCompositeOperation = 'source-over';

// Labels
ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.textAlign = 'center';
ctx.fillText('Red + Green = Yellow', 250, 310);
ctx.fillText('Red + Blue = Magenta', 250, 330);
ctx.fillText('Green + Blue = Cyan', 250, 350);
ctx.fillText('All three = White', 250, 370);
ctx.textAlign = 'left';`;

## Color Formats Canvas Understands

<p>Canvas accepts any color format that CSS understands. Let's explore each one so you know your options.</p>

<Concept>
  <p><strong>Named Colors:</strong> CSS has 140+ built-in color names like <code>crimson</code>, <code>cornflowerblue</code>, <code>salmon</code>. Easy to read, but limited palette.</p>
  <p><strong>Hex (#rrggbb):</strong> Each pair of hex digits represents a channel: <code>#ff0000</code> = red 255, green 0, blue 0. Compact, widely used.</p>
  <p><strong>RGB (r, g, b):</strong> <code>rgb(255, 0, 0)</code>, same as hex but easier to read. Values 0-255 per channel.</p>
  <p><strong>RGBA (r, g, b, a):</strong> Like RGB but with an alpha (opacity) channel from 0.0 (invisible) to 1.0 (solid).</p>
  <p><strong>HSL (h, s%, l%):</strong> Hue (0-360 degrees on a color wheel), Saturation (0-100%), Lightness (0-100%). Often more intuitive; we'll explore this next.</p>
</Concept>

## The HSL Color Model: Your Secret Weapon

<p>RGB makes you juggle three numbers to get a color. <strong>HSL (Hue, Saturation, Lightness)</strong> lets you think about color the way you already do: pick a color, decide how bold it should be, then choose how light or dark.</p>

<Concept>
  <p><strong>Hue</strong>: "What color?" Pick a spot on the rainbow. It is a number from 0 to 360, like degrees on a circle. 0 is red, 120 is green, 240 is blue, and 360 wraps back to red.</p>
  <p><strong>Saturation</strong>: "How vivid?" At 100% the color is pure and vibrant. At 0% all the color drains out and you get gray.</p>
  <p><strong>Lightness</strong>: "How light or dark?" 0% is always black, 100% is always white, and 50% gives you the true color.</p>
  <pre>{`
            The Hue Color Wheel (degrees)

                    0/360 = Red
                       |
            330       |       30
       Magenta  \\     |     /  Orange
                 \\    |    /
        300 ------+-------+----- 60
        Violet    |   *   |    Yellow
                 /    |    \\
            270 /     |     \\ 90
       Blue-Violet    |      Yellow-Green
                      |
                  180 = Cyan
                   (turquoise)

  Saturation: center (gray) ────> edge (vivid)
  Lightness:  separate axis: 0% black, 100% white`}</pre>
</Concept>

<p>HSL is incredibly useful in code because you can generate harmonious color schemes just by shifting the hue angle, and you can create tints and shades by adjusting lightness.</p>

export const hslWheelCode = `// Draw a visual hue wheel by painting thin slices
var centerX = 170;
var centerY = 190;
var outerRadius = 140;
var innerRadius = 80;

// Draw 360 slices, one per degree of hue
for (var angle = 0; angle < 360; angle++) {
  var startRad = (angle - 90) * Math.PI / 180;  // start at top
  var endRad = (angle - 89) * Math.PI / 180;

  ctx.fillStyle = 'hsl(' + angle + ', 100%, 50%)';
  ctx.beginPath();
  ctx.arc(centerX, centerY, outerRadius, startRad, endRad);
  ctx.arc(centerX, centerY, innerRadius, endRad, startRad, true);
  ctx.closePath();
  ctx.fill();
}

// Labels for key hues
ctx.fillStyle = 'silver';
ctx.font = '12px monospace';
ctx.textAlign = 'center';
ctx.fillText('0 Red', centerX, centerY - outerRadius - 8);
ctx.fillText('180 Cyan', centerX, centerY + outerRadius + 16);
ctx.fillText('90', centerX + outerRadius + 18, centerY + 4);
ctx.fillText('270', centerX - outerRadius - 18, centerY + 4);

// Show saturation/lightness variations on the right
var startX = 350;
ctx.textAlign = 'left';
ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('Saturation:', startX, 30);

// Same hue (210 = blue), varying saturation
for (var s = 0; s <= 100; s += 20) {
  var x = startX + (s / 100) * 120;
  ctx.fillStyle = 'hsl(210, ' + s + '%, 50%)';
  ctx.fillRect(x, 40, 22, 22);
}
ctx.fillStyle = 'gray';
ctx.font = '10px monospace';
ctx.fillText('0%              100%', startX, 76);

ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('Lightness:', startX, 105);

// Same hue, varying lightness
for (var l = 0; l <= 100; l += 10) {
  var lx = startX + (l / 100) * 120;
  ctx.fillStyle = 'hsl(210, 100%, ' + l + '%)';
  ctx.fillRect(lx, 115, 13, 22);
}
ctx.fillStyle = 'gray';
ctx.font = '10px monospace';
ctx.fillText('0%              100%', startX, 151);

// Show a complementary color scheme
ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('Complementary:', startX, 185);
ctx.fillText('(hue + 180)', startX, 200);

var baseHue = 210;
ctx.fillStyle = 'hsl(' + baseHue + ', 80%, 55%)';
ctx.fillRect(startX, 215, 50, 35);
ctx.fillStyle = 'hsl(' + (baseHue + 180) + ', 80%, 55%)';
ctx.fillRect(startX + 65, 215, 50, 35);

ctx.fillStyle = 'gray';
ctx.font = '10px monospace';
ctx.fillText('210', startX + 15, 265);
ctx.fillText('30', startX + 80, 265);

// Triadic scheme
ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('Triadic:', startX, 295);
ctx.fillText('(hue +120 each)', startX, 310);

for (var t = 0; t < 3; t++) {
  ctx.fillStyle = 'hsl(' + (baseHue + t * 120) + ', 80%, 55%)';
  ctx.fillRect(startX + t * 45, 320, 35, 35);
}
ctx.textAlign = 'left';`;

## fillStyle and strokeStyle

<p>Every shape you draw on Canvas uses two style properties. Think of them like choosing your art supplies:</p>

<Concept>
  <p><strong>fillStyle</strong>: the paint color that fills the inside of shapes. Like choosing which paint to pour into a bucket.</p>
  <p><strong>strokeStyle</strong>: the pen color for outlines. Like choosing which marker to trace edges with.</p>
  <p>Both accept any color format. Once set, they stay set until you change them, like picking up a new paintbrush.</p>
</Concept>

export const fillVsStrokeCode = `// fillStyle = interior color (the paint)
ctx.fillStyle = 'cyan';
ctx.fillRect(30, 30, 120, 100);
ctx.fillStyle = 'silver';
ctx.font = '13px sans-serif';
ctx.fillText('fillStyle', 55, 85);

// strokeStyle = outline color (the pen)
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 4;
ctx.strokeRect(190, 30, 120, 100);
ctx.fillStyle = 'silver';
ctx.fillText('strokeStyle', 207, 85);

// Both together
ctx.fillStyle = 'limegreen';
ctx.fillRect(350, 30, 120, 100);
ctx.strokeStyle = 'goldenrod';
ctx.lineWidth = 4;
ctx.strokeRect(350, 30, 120, 100);
ctx.fillStyle = 'silver';
ctx.fillText('both', 390, 85);

// Styles persist until changed
ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('Styles persist! Once set, every shape uses', 30, 175);
ctx.fillText('that style until you explicitly change it.', 30, 195);

// Show persistence
ctx.fillStyle = 'cyan';
ctx.fillRect(30, 220, 40, 40);  // blue
ctx.fillRect(80, 220, 40, 40);  // still blue!
ctx.fillRect(130, 220, 40, 40); // still blue!
ctx.fillStyle = 'deeppink';      // NOW we change it
ctx.fillRect(180, 220, 40, 40); // pink
ctx.fillRect(230, 220, 40, 40); // still pink

ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('fillStyle = blue', 30, 280);
ctx.fillText('change!', 180, 280);`;

## Transparency and Opacity

<p>Transparency lets you see through shapes to whatever is behind them. Canvas gives you two ways to control this.</p>

<Concept>
  <p><strong>Analogy: Glass Windows.</strong> Imagine stacking colored glass windows:</p>
  <pre>{`
  alpha = 1.0         alpha = 0.5          alpha = 0.0
  (fully opaque)      (semi-transparent)   (invisible)
  +──────────+        +──────────+         +──────────+
  │##########│        │# # # # # │         │          │
  │##########│        │ # # # #  │         │          │
  │## SOLID ##│        │# SEMI  # │         │ NOTHING  │
  │##########│        │ # # # #  │         │          │
  +──────────+        +──────────+         +──────────+
  Like a wall.        Like tinted          Like clear
  Blocks everything   glass. Lets some     glass. Fully
  behind it.          light through.       transparent.`}</pre>
  <p><strong>Method 1:</strong> <code>ctx.globalAlpha = 0.5</code> affects ALL subsequent draws. Remember to reset it to 1.0!</p>
  <p><strong>Method 2:</strong> <code>rgba(r, g, b, alpha)</code> or <code>hsla(h, s%, l%, alpha)</code> for per-color transparency. More precise control.</p>
</Concept>

export const alphaComparisonCode = `// === Left side: globalAlpha ===
ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('globalAlpha (affects everything)', 20, 25);

// Draw a base shape
ctx.fillStyle = 'cyan';
ctx.fillRect(30, 40, 180, 60);

// Now draw overlapping shapes with decreasing alpha
var alphas = [0.8, 0.5, 0.3, 0.1];
alphas.forEach(function(a, i) {
  ctx.globalAlpha = a;
  ctx.fillStyle = 'deeppink';
  ctx.fillRect(50 + i * 35, 60, 80, 120);
  // Even the label text is affected by globalAlpha!
});

ctx.globalAlpha = 1; // ALWAYS reset!
ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('0.8   0.5   0.3   0.1', 55, 200);

// === Right side: RGBA per-color ===
ctx.fillStyle = 'silver';
ctx.font = 'bold 13px sans-serif';
ctx.fillText('RGBA (per-color control)', 280, 25);

// Base shape
ctx.fillStyle = 'cyan';
ctx.fillRect(290, 40, 180, 60);

// Overlapping with per-color alpha: labels stay fully opaque!
var rgbaAlphas = [0.8, 0.5, 0.3, 0.1];
rgbaAlphas.forEach(function(a, i) {
  ctx.fillStyle = 'rgba(247, 120, 186, ' + a + ')';
  ctx.fillRect(310 + i * 35, 60, 80, 120);
});
// Notice: no need to reset anything!

ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('0.8   0.5   0.3   0.1', 315, 200);

// Warning note
ctx.fillStyle = 'goldenrod';
ctx.font = 'bold 12px sans-serif';
ctx.fillText('Warning: always reset globalAlpha!', 20, 240);
ctx.fillStyle = 'gray';
ctx.font = '12px monospace';
ctx.fillText('ctx.globalAlpha = 1;  // put this after', 20, 260);
ctx.fillText('Prefer rgba() for per-shape transparency.', 20, 280);`;

## Linear Gradients

<p>A gradient smoothly transitions between two or more colors. A <strong>linear gradient</strong> transitions along a straight line. Think of it like a paint roller that starts with one color and gradually shifts to another.</p>

<Concept>
  <p><code>createLinearGradient(x0, y0, x1, y1)</code> defines the line along which colors change:</p>
  <pre>{`
  Horizontal:  (0, 100) ─────────> (500, 100)
               start                end
               Left to right, same y

  Vertical:    (100, 0)
                  │
                  │  Top to bottom, same x
                  v
               (100, 400)

  Diagonal:    (0, 0)
                  \\
                   \\  Corner to corner
                    v
                   (500, 400)`}</pre>
  <p>After creating the gradient, add color stops: <code>gradient.addColorStop(position, color)</code></p>
  <p><code>position</code> is 0.0 (start) to 1.0 (end). You can add as many stops as you want!</p>
</Concept>

export const linearGradientBasicCode = `// === Horizontal gradient (left to right) ===
var g1 = ctx.createLinearGradient(20, 0, 480, 0);
g1.addColorStop(0, 'cyan');
g1.addColorStop(1, 'deeppink');
ctx.fillStyle = g1;
ctx.fillRect(20, 15, 460, 60);
ctx.fillStyle = 'silver';
ctx.font = '12px monospace';
ctx.fillText('horizontal: same y, different x', 140, 50);

// === Vertical gradient (top to bottom) ===
var g2 = ctx.createLinearGradient(0, 95, 0, 175);
g2.addColorStop(0, 'cyan');
g2.addColorStop(1, 'deeppink');
ctx.fillStyle = g2;
ctx.fillRect(20, 95, 460, 60);
ctx.fillStyle = 'silver';
ctx.fillText('vertical: same x, different y', 150, 130);

// === Diagonal gradient (corner to corner of a square) ===
var g3 = ctx.createLinearGradient(20, 175, 200, 355);
g3.addColorStop(0, 'cyan');
g3.addColorStop(1, 'deeppink');
ctx.fillStyle = g3;
ctx.fillRect(20, 175, 180, 180);
ctx.fillStyle = 'silver';
ctx.fillText('diagonal:', 55, 270);
ctx.fillText('both x AND y change', 35, 290);

// === Multi-stop gradient ===
var g4 = ctx.createLinearGradient(220, 0, 480, 0);
g4.addColorStop(0, 'deeppink');
g4.addColorStop(0.25, 'goldenrod');
g4.addColorStop(0.5, 'limegreen');

ctx.fillRect(220, 175, 260, 180);
ctx.fillStyle = 'black';
ctx.fillText('multi-stop: 5 colors', 270, 270);

// Label
ctx.fillStyle = 'gray';
ctx.font = '12px monospace';
ctx.fillText('Gradient direction = line from (x0,y0) to (x1,y1)', 80, 380);`;

## Radial Gradients

<p>A <strong>radial gradient</strong> transitions colors outward from a center point, like ripples in a pond. It's defined by two circles: an inner circle (where the gradient starts) and an outer circle (where it ends).</p>

<Concept>
  <p><code>createRadialGradient(x0, y0, r0, x1, y1, r1)</code></p>
  <pre>{`
  (x0, y0, r0) = inner circle center + radius  (gradient START)
  (x1, y1, r1) = outer circle center + radius  (gradient END)

  Centered (same center):        Off-center (spotlight):
  +───────────────────+          +───────────────────+
  │                   │          │                   │
  │     .-"""-.       │          │  .-"""-.          │
  │   /  inner  \\     │          │ / inner \\  outer  │
  │  │   (r0)   │     │          │ │  (r0)   \\  (r1) │
  │   \\        /      │          │  \\       / ---'   │
  │     '-...-'       │          │   '-...-'         │
  │    outer (r1)     │          │                   │
  +───────────────────+          +───────────────────+
  Colors radiate evenly          Light source is offset
  from center outward            creating a spotlight`}</pre>
</Concept>

  <p><strong>image</strong> = any image source (Image, Canvas, Video)</p>

## Composite Operations (Blending Modes)

<p><code>globalCompositeOperation</code> controls how new shapes interact with existing content on the canvas. The default is <code>"source-over"</code>, where new shapes paint on top. But there are 26 different modes that let you multiply, screen, subtract, and more.</p>

<Concept>
  <p><strong>Analogy: Photo editing layers.</strong> If you've used Photoshop or GIMP, composite operations are like blend modes. They control how a new layer combines with layers below it.</p>
  <p><strong>"source"</strong> = the new shape you're drawing</p>
  <p><strong>"destination"</strong> = what's already on the canvas</p>
  <pre>{`
  source-over      source-atop      destination-over     xor
  +------+         +------+         +------+             +------+
  |  BB  |         |  .B  |         |  AA  |             |  BA  |
  | AABB |         | AABB |         | AABB |             | AA.BB|
  | AA   |         | AA   |         | AABB |             | AA   |
  +------+         +------+         +------+             +------+
  New on top       New only where   Old on top           Overlap
  of old           old exists       of new               becomes clear

  A = existing (destination)     B = new shape (source)`}</pre>
</Concept>

export const compositeOpsCode = `// Show the most useful composite operations
var ops = [
  'source-over', 'source-atop', 'source-in', 'source-out',
  'destination-over', 'destination-atop', 'lighter', 'xor',
  'multiply', 'screen', 'overlay', 'difference'
];

var cols = 4;
var cellW = 110;
var cellH = 95;
var startX = 15;
var startY = 10;

ops.forEach(function(op, i) {
  var col = i % cols;
  var row = Math.floor(i / cols);
  var x = startX + col * (cellW + 10);
  var y = startY + row * (cellH + 15);

  // Background
  ctx.fillStyle = 'black';
  ctx.fillRect(x, y, cellW, cellH);

  // Save state for compositing
  ctx.save();

  // Clip to this cell
  ctx.beginPath();
  ctx.rect(x, y, cellW, cellH);
  ctx.clip();

  // Draw destination (blue circle) first
  ctx.globalCompositeOperation = 'source-over';
  ctx.fillStyle = 'cyan';
  ctx.beginPath();
  ctx.arc(x + 38, y + 45, 28, 0, Math.PI * 2);
  ctx.fill();

  // Apply the composite operation
  ctx.globalCompositeOperation = op;

  // Draw source (red circle)
  ctx.fillStyle = 'deeppink';
  ctx.beginPath();
  ctx.arc(x + 68, y + 45, 28, 0, Math.PI * 2);
  ctx.fill();

  ctx.restore();

  // Label
  ctx.globalCompositeOperation = 'source-over';
  ctx.fillStyle = 'gray';
  ctx.font = '9px monospace';
  ctx.fillText(op, x + 5, y + cellH + 10);
});

// Reset
ctx.globalCompositeOperation = 'source-over';`;

## Shadows

  { x: 10, w: 50, h: 160 },

  { x: 175, w: 45, h: 220 },

  bGrad.addColorStop(1, '#1a1a25');
  ctx.fillStyle = bGrad;
  ctx.fillRect(b.x, 250 - b.h, b.w, b.h);

  // Windows (randomly lit)
  for (var wy = 250 - b.h + 10; wy < 245; wy += 14) {
    for (var wx = b.x + 5; wx < b.x + b.w - 8; wx += 10) {

        var warmth = Math.random();
        ctx.fillStyle = warmth > 0.5

          : 'rgba(100, 180, 255, ' + (0.2 + warmth * 0.3) + ')';

  <p><strong>Lesson 3 Recap: What you learned.</strong></p>
  <p>Colors: RGB (how screens work), HSL (how humans think), and all CSS formats. <code>fillStyle</code> for fills, <code>strokeStyle</code> for outlines.</p>
  <p>Transparency: <code>globalAlpha</code> (affects everything, so always reset!) vs <code>rgba()</code>/<code>hsla()</code> (per-color control).</p>
  <p>Gradients: <code>createLinearGradient</code> for straight-line transitions, <code>createRadialGradient</code> for circular transitions. Both use <code>addColorStop()</code>.</p>
  <p>Patterns: <code>createPattern</code> repeats a tile image to fill shapes.</p>
  <p>Compositing: <code>globalCompositeOperation</code> controls how new shapes blend with existing content (26 modes!).</p>
  <p>Lines: <code>lineWidth</code>, <code>lineCap</code>, <code>lineJoin</code>, <code>setLineDash</code>, and animated <code>lineDashOffset</code>.</p>
  <p>Shadows: <code>shadowColor</code>, <code>shadowBlur</code>, <code>shadowOffsetX/Y</code>, used for glow, depth, and drop shadows.</p>
