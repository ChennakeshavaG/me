Rectangles are useful for prototyping, but real graphics need triangles, circles, stars, and freeform shapes. This lesson introduces the **path system**: Canvas's way of describing any shape you can imagine. By the end, you will draw polygons, smooth curves, and combine them into complex figures.

## The "Pen" Metaphor

<p>In Lesson 1, we drew rectangles, but the real world isn't made of rectangles. To draw triangles, circles, arrows, speech bubbles, or any custom shape, we need <strong>paths</strong>.</p>

<p>Imagine you're holding an invisible pen over a piece of paper:</p>

<Concept>
  <p><strong>Drawing with an invisible pen:</strong></p>
  <pre>{`    1. Pick up the pen and place it somewhere
       --> moveTo(x, y)         "Move without drawing"

    2. Drag the pen to draw a line
       --> lineTo(x, y)         "Draw a straight line to here"

    3. Draw a curve
       --> arc(x, y, r, startAngle, endAngle)
       --> bezierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)

    4. Go back to where you started
       --> closePath()           "Connect back to start"

    5. Fill in the shape or trace its outline
       --> fill() or stroke()   "Apply the ink"

    The pen REMEMBERS where it is after each command.
    Every new command starts from where the last one ended.`}</pre>
</Concept>

<p>The key insight: <strong>path commands don't draw anything visible</strong>. They build up an invisible outline. Only when you call <code>fill()</code> or <code>stroke()</code> does the shape actually appear on the canvas.</p>

## Why beginPath() is Critical

<p><code>beginPath()</code> starts a brand new path, throwing away whatever path was being built before. This is the <strong>single most common source of canvas bugs for beginners</strong>: forgetting <code>beginPath()</code>.</p>

export const beginpathBugCode = `// ---- BUG: No beginPath() between shapes ----
// The left side shows what happens when you FORGET beginPath()

ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('BUG: forgot beginPath()', 20, 25);

ctx.beginPath();
ctx.moveTo(50, 50);
ctx.lineTo(150, 50);
ctx.lineTo(100, 130);
ctx.closePath();
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 2;
ctx.stroke();

// Oops! No beginPath() here: the old triangle is still in the path!
ctx.moveTo(50, 160);
ctx.lineTo(150, 160);
ctx.lineTo(100, 240);
ctx.closePath();
ctx.strokeStyle = 'cyan';
ctx.lineWidth = 2;
ctx.stroke();
// BOTH triangles get stroked blue!

// ---- FIX: Always use beginPath() ----
ctx.fillStyle = 'gray';
ctx.fillText('FIX: use beginPath()', 280, 25);

ctx.beginPath();  // fresh start
ctx.moveTo(310, 50);
ctx.lineTo(410, 50);
ctx.lineTo(360, 130);
ctx.closePath();
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 2;
ctx.stroke();

ctx.beginPath();  // another fresh start!
ctx.moveTo(310, 160);
ctx.lineTo(410, 160);
ctx.lineTo(360, 240);
ctx.closePath();
ctx.strokeStyle = 'cyan';
ctx.lineWidth = 2;
ctx.stroke();
// Each triangle has its own color, correct!`;

## moveTo and lineTo: Straight Lines

<p><code>moveTo(x, y)</code> picks up the pen and places it at a new position without drawing. <code>lineTo(x, y)</code> draws a straight line from wherever the pen currently is to the new point.</p>

export const linesCode = `// Draw a simple triangle
ctx.beginPath();
ctx.moveTo(250, 30);    // start at top center
ctx.lineTo(100, 250);   // line down to bottom-left
ctx.lineTo(400, 250);   // line across to bottom-right
ctx.closePath();         // line back to start (top center)

ctx.strokeStyle = 'cyan';
ctx.lineWidth = 3;
ctx.stroke();

// Label the points

ctx.font = '12px monospace';

ctx.fillText('lineTo(100, 250)', 10, 265);

// Show the pen path with dotted arrows

ctx.stroke();
ctx.setLineDash([]);`;

## closePath: Connecting Back to Start

<p><code>closePath()</code> draws a straight line from the current pen position back to the very first point of the current path (where you called <code>moveTo()</code> initially). It's how you close a shape cleanly.</p>

export const closePathCode = `// WITHOUT closePath: notice the gap at bottom-left
ctx.beginPath();
ctx.moveTo(50, 50);
ctx.lineTo(200, 50);
ctx.lineTo(200, 200);
ctx.lineTo(50, 200);
// no closePath: the bottom-left corner is open!
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 3;
ctx.stroke();

ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('no closePath()', 60, 240);
ctx.fillText('gap here!', 10, 140);
// Arrow pointing to gap
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 1;
ctx.beginPath();
ctx.moveTo(55, 135);
ctx.lineTo(48, 115);
ctx.stroke();

// WITH closePath: clean closed shape
ctx.beginPath();
ctx.moveTo(280, 50);
ctx.lineTo(430, 50);
ctx.lineTo(430, 200);
ctx.lineTo(280, 200);
ctx.closePath();         // draws line back to (280,50)
ctx.strokeStyle = 'limegreen';
ctx.lineWidth = 3;
ctx.stroke();

ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('with closePath()', 290, 240);`;

  <p><strong>Important note:</strong> <code>fill()</code> automatically closes the path for you (it has to, since you can't fill an open shape). But <code>stroke()</code> does NOT auto-close. </Concept>

## fill() vs stroke(): Filling vs Outlining

<p>After building a path, you have two choices for making it visible:</p>
<ul>
  <li><code>fill()</code> colors in the entire interior of the shape</li>
  <li><code>stroke()</code> draws just the outline</li>
</ul>
<p>You can use both on the same path!</p>

export const fillVsStrokeCode = `// Helper: draw a pentagon at (cx, cy)
function pentagon(cx, cy, r) {
  ctx.beginPath();
  for (var i = 0; i < 5; i++) {
    var angle = (i * 2 * Math.PI / 5) - Math.PI / 2;
    var px = cx + r * Math.cos(angle);
    var py = cy + r * Math.sin(angle);
    if (i === 0) ctx.moveTo(px, py);
    else ctx.lineTo(px, py);
  }
  ctx.closePath();
}

// FILL only
pentagon(90, 140, 65);
ctx.fillStyle = 'cyan';
ctx.fill();
ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('fill()', 55, 240);

// STROKE only
pentagon(250, 140, 65);
ctx.strokeStyle = 'deeppink';
ctx.lineWidth = 3;
ctx.stroke();
ctx.fillStyle = 'gray';
ctx.fillText('stroke()', 210, 240);

// BOTH fill and stroke
pentagon(410, 140, 65);
ctx.fillStyle = 'limegreen';
ctx.fill();
ctx.strokeStyle = 'silver';
ctx.lineWidth = 3;
ctx.stroke();
ctx.fillStyle = 'gray';
ctx.fillText('fill + stroke', 355, 240);

ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('Tip: call fill() BEFORE stroke(),', 120, 280);
ctx.fillText('so the stroke appears on top of the fill.', 105, 296);`;

## Arcs and Circles

<p><code>arc(x, y, radius, startAngle, endAngle, counterclockwise)</code> draws a circular arc (a portion of a circle, or the whole circle). Angles are in <strong>radians</strong>, not degrees.</p>

<Concept>
  <p><strong>Radians cheat sheet:</strong></p>
  <pre>{`    Degrees     Radians            Position on clock
    ───────     ─────────────      ──────────────────
    0           0                  3 o'clock (right)
    90          Math.PI / 2        6 o'clock (bottom)
    180         Math.PI            9 o'clock (left)
    270         Math.PI * 1.5      12 o'clock (top)
    360         Math.PI * 2        full circle

    Formula: radians = degrees * Math.PI / 180

                    270 / PI*1.5
                         │
                         │
    180 / PI  ───────────●─────────── 0 (start)
                         │
                         │
                    90 / PI/2

    REMEMBER: 0 radians points RIGHT, not up!
    And angles go CLOCKWISE (Y is flipped in canvas).`}</pre>
</Concept>

export const arcsCode = `// Full circle
ctx.beginPath();
ctx.arc(80, 80, 50, 0, Math.PI * 2);
ctx.fillStyle = 'cyan';
ctx.fill();
ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('full circle', 42, 145);
ctx.fillText('0 to PI*2', 45, 160);

// Half circle (top half, from PI to 0)
ctx.beginPath();
ctx.arc(220, 80, 50, Math.PI, 0);
ctx.fillStyle = 'deeppink';
ctx.fill();
ctx.fillStyle = 'gray';
ctx.fillText('semicircle', 182, 145);
ctx.fillText('PI to 0', 195, 160);

// Quarter arc (stroke only)
ctx.beginPath();
ctx.arc(360, 80, 50, 0, Math.PI / 2);
ctx.strokeStyle = 'limegreen';

ctx.stroke();
ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('quarter arc', 322, 145);
ctx.fillText('0 to PI/2', 330, 160);

// Pac-man: use moveTo to center, then arc, then closePath
ctx.beginPath();

ctx.arc(80, 250, 50, 0.3, Math.PI * 2 - 0.3);

ctx.fillStyle = 'goldenrod';
ctx.fill();

ctx.fillText('pac-man', 48, 320);

// Ring (thick stroked circle)
ctx.beginPath();

// Smiley face

ctx.fillText('combine arcs', 340, 320);`;

## Arc Direction: Clockwise vs Counterclockwise

<p>The last parameter of <code>arc()</code> is <code>counterclockwise</code> (default: <code>false</code> = clockwise). This controls <em>which way</em> the arc sweeps from the start angle to the end angle.</p>

export const arcDirectionCode = `var startAngle = 0;
var endAngle = Math.PI * 1.2;

// Clockwise (default, counterclockwise = false)
ctx.beginPath();
ctx.arc(140, 130, 80, startAngle, endAngle, false);
ctx.lineTo(140, 130);   // line to center
ctx.closePath();
ctx.fillStyle = 'cyan';
ctx.fill();
ctx.strokeStyle = 'silver';
ctx.lineWidth = 2;
ctx.stroke();

ctx.fillStyle = 'gray';
ctx.font = '13px monospace';
ctx.fillText('clockwise (default)', 52, 240);
ctx.fillText('false', 110, 260);

// Counterclockwise (counterclockwise = true)
ctx.beginPath();
ctx.arc(360, 130, 80, startAngle, endAngle, true);
ctx.lineTo(360, 130);   // line to center
ctx.closePath();
ctx.fillStyle = 'deeppink';
ctx.fill();
ctx.strokeStyle = 'silver';
ctx.lineWidth = 2;
ctx.stroke();

ctx.fillStyle = 'gray';
ctx.fillText('counterclockwise', 285, 240);
ctx.fillText('true', 340, 260);

// Show the start/end angles
ctx.fillStyle = 'limegreen';
ctx.font = '11px monospace';
ctx.fillText('Same start & end angles,', 140, 20);
ctx.fillText('different sweep direction!', 140, 36);`;

<Concept>
  <p><strong>When does direction matter?</strong> For a full circle (0 to PI*2), it doesn't matter. But for partial arcs (pie charts, gauges, pac-man), the direction determines which "slice" of the circle you get. Same start and end angle, but clockwise gives you the big slice while counterclockwise gives the small slice (or vice versa).</p>
</Concept>

## rect(): Rectangle as a Path

<p>We used <code>fillRect()</code> and <code>strokeRect()</code> in Lesson 1, but there's also <code>rect(x, y, width, height)</code> which adds a rectangle to the current <em>path</em> without drawing it immediately. This lets you combine rectangles with other path operations.</p>

export const rectPathCode = `// Build a path with multiple rectangles

## ellipse(): Ovals and Rotated Circles

<p><code>ellipse(x, y, radiusX, radiusY, rotation, startAngle, endAngle)</code> is like <code>arc()</code> but allows different horizontal and vertical radii, plus rotation. The egg example below also uses <code>save()</code>, <code>translate()</code>, <code>scale()</code>, and <code>restore()</code> to stretch a circle into an oval; these transform methods will be covered in a later lesson.</p>

ctx.beginPath();

ctx.fillStyle = 'cyan';
ctx.fill();
ctx.fillStyle = 'gray';
ctx.font = '11px monospace';
ctx.fillText('radiusX=70', 55, 165);
ctx.fillText('radiusY=40', 55, 180);

// A tall oval
ctx.beginPath();
ctx.ellipse(260, 100, 35, 70, 0, 0, Math.PI * 2);
ctx.fillStyle = 'deeppink';
ctx.fill();
ctx.fillStyle = 'gray';
ctx.fillText('radiusX=35', 218, 185);
ctx.fillText('radiusY=70', 218, 200);

// A rotated ellipse (45 degrees)
ctx.beginPath();
ctx.ellipse(420, 100, 70, 30, Math.PI / 4, 0, Math.PI * 2);

ctx.fill();

ctx.fillText('egg (scaled arc)', 130, 255);`;

## Quadratic Curves: One Control Point

<p><code>quadraticCurveTo(cpx, cpy, endX, endY)</code> draws a curve from the current pen position, bending toward a <strong>control point</strong>, and ending at the endpoint.</p>

<Concept>
  <p><strong>The rubber band analogy:</strong></p>
  <pre>{`    Imagine a rubber band pinned at START and END.
    The control point is a finger pulling the band sideways.

    control point (cpx, cpy)
         ●
        / \\
       /   \\         The curve bends TOWARD
      /     \\        the control point but
     /       \\       never actually touches it
    ●─────────●      (usually).
   START      END

    The further the control point is from
    the straight line, the more the curve bends.`}</pre>
</Concept>

<Concept variant="note">
**Want the actual math?** The quadratic Bezier formula is `B(t) = (1-t)^2*P0 + 2*(1-t)*t*P1 + t^2*P2`. It's just two levels of lerp (linear interpolation) stacked together. See the <a href="/wiki/canvas-lab/appendix-bezier-math">Math Appendix</a> for the full derivation, interactive visualizations, and why it works.
</Concept>

export const quadraticCode = `var startX = 50, startY = 250;
var cpX = 250, cpY = 30;      // control point
var endX = 450, endY = 250;

// Draw the curve
ctx.beginPath();
ctx.moveTo(startX, startY);
ctx.quadraticCurveTo(cpX, cpY, endX, endY);
ctx.strokeStyle = 'cyan';
ctx.lineWidth = 3;
ctx.stroke();

// Show control point and guide lines (dashed)
ctx.setLineDash([5, 5]);
ctx.strokeStyle = '#1a1a2a';
ctx.lineWidth = 1;
ctx.beginPath();
ctx.moveTo(startX, startY);
ctx.lineTo(cpX, cpY);
ctx.lineTo(endX, endY);
ctx.stroke();
ctx.setLineDash([]);

// Label the three points
ctx.fillStyle = 'limegreen';
ctx.beginPath(); ctx.arc(startX, startY, 6, 0, Math.PI * 2); ctx.fill();
ctx.beginPath(); ctx.arc(endX, endY, 6, 0, Math.PI * 2); ctx.fill();
ctx.fillStyle = 'deeppink';
ctx.beginPath(); ctx.arc(cpX, cpY, 6, 0, Math.PI * 2); ctx.fill();

ctx.font = '12px monospace';
ctx.fillStyle = 'limegreen';
ctx.fillText('start', startX - 5, startY + 20);
ctx.fillText('end', endX - 15, endY + 20);
ctx.fillStyle = 'deeppink';
ctx.fillText('control point', cpX - 35, cpY - 12);
ctx.fillText('(pulls the curve)', cpX - 45, cpY + 20);`;

## Bezier Curves: Two Control Points

<p><code>bezierCurveTo(cp1x, cp1y, cp2x, cp2y, endX, endY)</code> is the more powerful cousin of quadratic curves. With <strong>two</strong> control points, you can make S-curves, loops, and the smooth curves used in fonts and vector graphics.</p>

<Concept>
  <p><strong>How two control points shape a curve:</strong></p>
  <pre>{`    CP1 controls the      CP2 controls the
    START of the curve    END of the curve
        ●                     ●
       ╱                       ╲
      ╱   The curve leaves      ╲
     ╱    START heading          ╲
    ●     toward CP1, then   ────●
   START  bends to arrive       END
          from CP2's direction

    Think of it as: CP1 says "leave this way"
                    CP2 says "arrive this way"

    S-curve (control points on opposite sides):
     ● CP1
      ╲
       ╲    ╱────╲
    ●───╲──╱      ╲──●
   START            END
                  ╱
                 ╱
                ● CP2`}</pre>
</Concept>

<Concept variant="note">
**The cubic formula:** `B(t) = (1-t)^3*P0 + 3*(1-t)^2*t*P1 + 3*(1-t)*t^2*P2 + t^3*P3`. Three levels of lerp. The <a href="/wiki/canvas-lab/appendix-bezier-math">Math Appendix</a> derives this step by step and shows why the coefficients 1, 3, 3, 1 come from Pascal's triangle.
</Concept>

export const bezierCode = `var sx = 50, sy = 150;       // start point
var cp1x = 150, cp1y = 30;   // control point 1
var cp2x = 350, cp2y = 280;  // control point 2
var ex = 450, ey = 150;      // end point

// The S-curve
ctx.beginPath();
ctx.moveTo(sx, sy);
ctx.bezierCurveTo(cp1x, cp1y, cp2x, cp2y, ex, ey);
ctx.strokeStyle = 'cyan';
ctx.lineWidth = 3;
ctx.stroke();

// Guide lines from start->cp1 and end->cp2
ctx.setLineDash([5, 5]);
ctx.strokeStyle = '#1a1a2a';
ctx.lineWidth = 1;
ctx.beginPath();
ctx.moveTo(sx, sy);
ctx.lineTo(cp1x, cp1y);
ctx.stroke();
ctx.beginPath();
ctx.moveTo(ex, ey);
ctx.lineTo(cp2x, cp2y);
ctx.stroke();
ctx.setLineDash([]);

// Label all four points
ctx.fillStyle = 'limegreen';
ctx.beginPath(); ctx.arc(sx, sy, 5, 0, Math.PI * 2); ctx.fill();
ctx.beginPath(); ctx.arc(ex, ey, 5, 0, Math.PI * 2); ctx.fill();
ctx.fillStyle = 'deeppink';
ctx.beginPath(); ctx.arc(cp1x, cp1y, 5, 0, Math.PI * 2); ctx.fill();
ctx.beginPath(); ctx.arc(cp2x, cp2y, 5, 0, Math.PI * 2); ctx.fill();

ctx.font = '12px monospace';
ctx.fillStyle = 'limegreen';
ctx.fillText('start', sx - 5, sy - 12);
ctx.fillText('end', ex - 10, ey - 12);
ctx.fillStyle = 'deeppink';
ctx.fillText('cp1 (pulls start)', cp1x + 8, cp1y);
ctx.fillText('cp2 (pulls end)', cp2x + 8, cp2y);`;

## Fill Rules: nonzero vs evenodd

<p>When paths overlap or have holes, how does Canvas decide what's "inside" the shape? That's the <strong>fill rule</strong>. You pass it to <code>fill()</code>.</p>

export const starConstructionCode = `// Watch how a star is drawn: skip-connecting pentagon vertices
var cx = 250, cy = 185, r = 110;

// Calculate all 5 pentagon vertices
var pts = [];
for (var i = 0; i < 5; i++) {
  var angle = (i * 2 * Math.PI / 5) - Math.PI / 2;
  pts.push({
    x: cx + r * Math.cos(angle),
    y: cy + r * Math.sin(angle),
    label: i
  });
}

// Star connection order: 0 -> 2 -> 4 -> 1 -> 3 -> 0
var order = [0, 2, 4, 1, 3];
var step = 0;
var progress = 0;
var speed = 0.02;
var pause = 0;

function draw() {
  ctx.clearRect(0, 0, w, h);

  // Draw faint circle showing where vertices sit
  ctx.strokeStyle = '#1a1a2a';
  ctx.lineWidth = 1;
  ctx.setLineDash([4, 4]);
  ctx.beginPath();
  ctx.arc(cx, cy, r, 0, Math.PI * 2);
  ctx.stroke();
  ctx.setLineDash([]);

  // Draw all 5 vertex dots and labels
  for (var i = 0; i < 5; i++) {
    var isCurrent = (step < 5 && order[step] === i);

    ctx.arc(pts[i].x, pts[i].y, isCurrent ? 7 : 5, 0, Math.PI * 2);
    ctx.fill();
    ctx.fillStyle = isCurrent ? 'limegreen' : 'silver';
    ctx.font = 'bold 14px monospace';
    var lx = pts[i].x + (pts[i].x > cx ? 12 : -20);
    var ly = pts[i].y + (pts[i].y > cy ? 20 : -10);
    ctx.fillText(i, lx, ly);
  }

  for (var s = 0; s < step && s < 5; s++) {
    var from = pts[order[s]];
    var to = pts[order[(s + 1) % 5]];
    ctx.strokeStyle = 'cyan';
    ctx.lineWidth = 2.5;
    ctx.beginPath();
    ctx.moveTo(from.x, from.y);
    ctx.lineTo(to.x, to.y);
    ctx.stroke();

  // Draw current line being animated
  if (step < 5) {
    var from = pts[order[step]];
    var to = pts[order[(step + 1) % 5]];
    var mx = from.x + (to.x - from.x) * progress;
    var my = from.y + (to.y - from.y) * progress;

    ctx.lineWidth = 3;
    ctx.beginPath();
    ctx.moveTo(from.x, from.y);
    ctx.lineTo(mx, my);
    ctx.stroke();

    ctx.fillStyle = 'limegreen';
    ctx.beginPath();
    ctx.arc(mx, my, 5, 0, Math.PI * 2);
    ctx.fill();

  ctx.font = '14px monospace';
  ctx.fillText('Connection order: 0 -> 2 -> 4 -> 1 -> 3 -> 0', 50, 25);
  ctx.fillText('(skip one vertex each time)', 130, 45);

    var toIdx = order[(step + 1) % 5];
    ctx.fillStyle = 'limegreen';
    ctx.fillText('Drawing: ' + fromIdx + ' -> ' + toIdx, 180, h - 15);

    ctx.fillText('All lines cross! Star complete.', 140, h - 15);

    if (progress >= 1) {
      progress = 0;
      step++;
    }
  } else {
    pause++;
    if (pause > 120) {
      step = 0;
      progress = 0;
      pause = 0;
    }

<p>How does alternating between two radii create teeth? The idea: place <strong>2 x teeth</strong> points around a circle. Even points sit on the <strong>outer</strong> radius (tooth tips), odd points sit on the <strong>inner</strong> radius (tooth valleys). Connect them in order and you get a gear.</p>

export const gearConstructionCode = `var gears = [
  { cx: 250, cy: 230, outerR: 120, innerR: 90, teeth: 12, color: 'cyan', label: '12-tooth gear' },
  { cx: 250, cy: 220, outerR: 80, innerR: 58, teeth: 8, color: 'deeppink', label: '8-tooth gear' }

var step = 0;
var progress = 0;
var speed = 0.02;
var pause = 0;

  ctx.clearRect(0, 0, w, h);
  var g = gears[activeGear];

  ctx.fillStyle = g.color;
  ctx.font = 'bold 14px monospace';
  ctx.fillText(g.label.toUpperCase(), 20, 22);
  ctx.fillStyle = 'silver';
  ctx.font = '12px monospace';

  ctx.font = '11px monospace';
  ctx.fillStyle = g.color;
  ctx.fillRect(370, 14, 8, 8);
  ctx.fillStyle = '#888';
  ctx.fillText('= tip', 384, 22);
  ctx.fillStyle = '#666';
  ctx.fillRect(370, 30, 8, 8);
  ctx.fillStyle = '#888';
  ctx.fillText('= valley', 384, 38);

  ctx.strokeStyle = '#1a1a2a';
  ctx.lineWidth = 1;
  ctx.beginPath();
  ctx.moveTo(20, 55);
  ctx.lineTo(w - 20, 55);
  ctx.stroke();

  ctx.setLineDash([6, 4]);
  ctx.lineWidth = 1.5;
  ctx.strokeStyle = '#4a4a5a';
  ctx.beginPath();
  ctx.arc(g.cx, g.cy, g.outerR, 0, Math.PI * 2);
  ctx.stroke();
  ctx.strokeStyle = '#3a3a4a';
  ctx.beginPath();
  ctx.arc(g.cx, g.cy, g.innerR, 0, Math.PI * 2);
  ctx.stroke();
  ctx.setLineDash([]);

  // Radius labels above the gear
  ctx.fillStyle = '#777';
  ctx.font = '11px monospace';
  ctx.fillText('outer r', g.cx - 22, g.cy - g.outerR - 8);
  ctx.fillText('inner r', g.cx - 22, g.cy - g.innerR + 16);

  var pts = [];
  for (var i = 0; i < totalPts; i++) {
    var r = (i % 2 === 0) ? g.outerR : g.innerR;
    var angle = (i * Math.PI / g.teeth) - Math.PI / 2;
    pts.push({
      x: g.cx + r * Math.cos(angle),
      y: g.cy + r * Math.sin(angle),
      isOuter: (i % 2 === 0)

  for (var s = 0; s < step && s < totalPts; s++) {
    var from = pts[s];

    ctx.beginPath();
    ctx.moveTo(from.x, from.y);
    ctx.lineTo(to.x, to.y);
    ctx.stroke();
    ctx.globalAlpha = 1;

  // Draw vertex dots
  for (var i = 0; i < totalPts; i++) {
    var drawn = (i <= step);
    var isCurrent = (step < totalPts && i === step);
    var isNext = (step < totalPts && i === (step + 1) % totalPts);
    if (!drawn && !isNext) continue;
    ctx.beginPath();
    ctx.arc(pts[i].x, pts[i].y, isCurrent ? 6 : 4, 0, Math.PI * 2);

    ctx.fill();

  if (step < totalPts) {

    ctx.beginPath();
    ctx.arc(mx, my, 4, 0, Math.PI * 2);
    ctx.fill();

  ctx.font = '12px monospace';

    var ptType = (step % 2 === 0) ? 'OUTER (tip)' : 'INNER (valley)';
    var nextType = ((step + 1) % 2 === 0) ? 'OUTER (tip)' : 'INNER (valley)';

    ctx.fillStyle = 'goldenrod';
    ctx.fillText('Next ' + ((step + 1) % totalPts) + ': ' + nextType, 20, h - 10);

    ctx.fillText('All ' + totalPts + ' points connected!', 20, h - 30);

    if (progress >= 1) {

      step++;
    }
  } else {

      progress = 0;
      pause = 0;

  var my = (e.clientY - rect.top) * scaleY;

  // Build the circle path again for hit testing
  ctx.beginPath();
  ctx.arc(circleX, circleY, circleR, 0, Math.PI * 2);

  if (ctx.isPointInPath(mx, my)) {
    message = 'HIT! You clicked inside the circle!';
    messageColor = 'limegreen';
  } else {
    message = 'MISS! Click was at (' + Math.round(mx) + ', ' + Math.round(my) + ')';
    messageColor = 'deeppink';
  }
  draw();
});`;

<Concept>
  <p><strong>How it works:</strong> After building a path (but before or after drawing it), call <code>ctx.isPointInPath(x, y)</code>. It returns <code>true</code> if the point is inside the path, <code>false</code> otherwise. This works for ANY shape: circles, polygons, custom paths, even bezier curves!</p>
</Concept>

<Concept>
  <p><strong>What you learned in this lesson:</strong></p>
  <pre>{`    Concept                How to do it
    ────────────────────── ─────────────────────────────────────
    Start a new path       ctx.beginPath()
    Move pen (no drawing)  ctx.moveTo(x, y)
    Straight line          ctx.lineTo(x, y)
    Close the shape        ctx.closePath()
    Fill the shape         ctx.fill()  or  ctx.fill('evenodd')
    Outline the shape      ctx.stroke()
    Circle/arc             ctx.arc(x, y, r, start, end, ccw?)
    Oval                   ctx.ellipse(x, y, rx, ry, rot, s, e)
    Rectangle path         ctx.rect(x, y, w, h)
    Simple curve           ctx.quadraticCurveTo(cpx, cpy, x, y)
    Complex curve          ctx.bezierCurveTo(c1x,c1y,c2x,c2y,x,y)
    Hit testing            ctx.isPointInPath(x, y)

    Key ideas:
    - ALWAYS call beginPath() before a new shape
    - Path commands are invisible: fill()/stroke() makes them real
    - fill() auto-closes paths; stroke() does not
    - Angles are in RADIANS (PI = 180 degrees)
    - Combine line, arc, and curve calls in one path for complex shapes`}</pre>
</Concept>
