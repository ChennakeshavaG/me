## What Is Animation?

<p>Animation is an illusion. Nothing actually "moves" on your screen. Instead, you show a series of still images so quickly that your brain perceives motion. It is exactly the same trick as a flipbook: each page has a slightly different drawing, and when you flip through them fast enough, things appear to move.</p>

<Concept>
  <p><strong>The flipbook analogy:</strong></p>
  <pre>{`  Frame 1       Frame 2       Frame 3       Frame 4
  ┌────────┐   ┌────────┐   ┌────────┐   ┌────────┐
  │        │   │        │   │        │   │        │
  │ ●      │   │  ●     │   │   ●    │   │    ●   │
  │        │   │        │   │        │   │        │
  └────────┘   └────────┘   └────────┘   └────────┘

  Show these fast enough → your brain sees a ball moving right!

  Canvas animation:
  1. Clear the canvas        (erase the page)
  2. Update positions        (draw the next frame slightly different)
  3. Draw everything         (show the new page)
  4. Repeat 60 times/second  (flip fast!)`}</pre>
</Concept>

<p>On a Canvas, we do this by writing a function that clears the screen, updates some numbers (positions, sizes, colors), draws everything fresh, and then asks the browser to call that function again for the next frame. That cycle (clear, update, draw, repeat) is called the <strong>game loop</strong> (even if you are not making a game).</p>

## Why 60 Frames Per Second?

<p>Most monitors refresh their display 60 times per second. That is 60 <em>frames per second</em>, or <strong>60fps</strong>. Movies use 24fps, which is enough for film but looks choppy for interactive graphics. At 60fps, each frame lasts about 16.7 milliseconds. If your drawing code takes longer than that, frames get dropped and animation looks janky.</p>

<Concept>
  <p><strong>Frame timing at different rates:</strong></p>
  <pre>{`  Framerate   Time per frame   Feels like
  ─────────   ──────────────   ──────────
   10 fps       100.0 ms       slideshow, painful
   24 fps        41.7 ms       cinema film
   30 fps        33.3 ms       okay for video
   60 fps        16.7 ms       smooth for interaction ← target
  120 fps         8.3 ms       butter-smooth (gaming monitors)

  Rule of thumb: keep your update+draw under 16ms and
  the browser handles the rest.`}</pre>
</Concept>

## requestAnimationFrame vs setInterval

<p>You might think: "I will just use <code>setInterval(draw, 16)</code> to run my loop 60 times per second." That works, but it has serious problems. <code>setInterval</code> is a general-purpose timer; it does not know anything about the screen. It will keep firing even when the browser tab is hidden (wasting battery), it drifts out of sync with the monitor, and it cannot adapt to different refresh rates.</p>

<p><code>requestAnimationFrame(callback)</code> was built specifically for animation. It syncs with the display refresh, pauses automatically when the tab is hidden, and lets the browser batch your drawing with other visual updates for maximum efficiency.</p>

<Concept>
  <p><strong>setInterval vs requestAnimationFrame:</strong></p>
  <pre>{`  setInterval(draw, 16):
  ├─ fires every ~16ms regardless of monitor refresh
  ├─ keeps firing when tab is hidden (wastes CPU/battery)
  ├─ can drift: 16ms timer ≠ exactly 60fps
  └─ no guarantee drawing happens before next screen paint

  requestAnimationFrame(draw):
  ├─ fires exactly once before each screen repaint
  ├─ pauses automatically when tab is hidden
  ├─ adapts to any refresh rate (60hz, 120hz, 144hz)
  └─ browser optimizes paint timing

  Always use requestAnimationFrame for canvas animation.`}</pre>
</Concept>

export const firstAnimCode = `// A ball moving across the screen: the simplest animation

var x = 0;  // ball's horizontal position

function loop() {
  // Step 1: Clear the entire canvas
  ctx.clearRect(0, 0, 500, 300);

  // Step 2: Update the position
  x += 2;                    // move 2 pixels to the right
  if (x > 550) x = -50;     // wrap around when off-screen

  // Step 3: Draw the ball at its new position
  ctx.fillStyle = 'cyan';
  ctx.beginPath();
  ctx.arc(x, 150, 25, 0, Math.PI * 2);
  ctx.fill();

  // Step 4: Ask for the next frame
  // Store the ID in __raf so the demo can cancel it on reset
  __raf = requestAnimationFrame(loop);
}

// Start the loop
loop();`;

<p>Notice the pattern: <code>clearRect</code> wipes the slate, we move <code>x</code> a bit, draw the circle at the new <code>x</code>, and call <code>requestAnimationFrame(loop)</code> to schedule the next frame. Without this, animation loops keep running forever, even after removing the canvas from the page, wasting CPU and causing memory leaks.</p>

## The Timestamp Parameter

<p>When the browser calls your animation function, it passes one argument: a <strong>timestamp</strong> in milliseconds. This is a high-precision number from <code>performance.now()</code> that tells you exactly when this frame started. You will use it to calculate how much time has passed between frames.</p>

<Concept>
  <pre>{`  function loop(timestamp) {
    // timestamp = 16834.5  (milliseconds since page load)
    // ...draw stuff...
    __raf = requestAnimationFrame(loop);
  }
  __raf = requestAnimationFrame(loop);
  //                              ↑
  // The browser calls loop(timestamp) for you.
  // You never pass the timestamp yourself: the browser does.`}</pre>
</Concept>

## Delta Time: Frame-Rate Independent Motion

<p>In the first demo, we moved the ball by a fixed 2 pixels every frame. That means on a 60fps monitor the ball moves 120 pixels per second, but on a 120fps monitor it moves 240 pixels per second, twice as fast! This is a bug, not a feature.</p>

<p>The fix is <strong>delta time</strong> (often written <code>dt</code>). Instead of moving a fixed amount per frame, you decide a speed in <em>pixels per second</em>, then multiply by how many seconds actually passed since the last frame.</p>

<Concept>
  <p><strong>The delta time formula:</strong></p>
  <pre>{`  speed = 150 pixels per second

  Frame at 60fps (dt = 0.0167s):
    movement = 150 * 0.0167 = 2.5 pixels this frame

  Frame at 120fps (dt = 0.0083s):
    movement = 150 * 0.0083 = 1.25 pixels this frame

  Both rates → 150 pixels per second. Consistent speed!

  Formula:  distance = speed * deltaTime
            where deltaTime = (now - lastTime) / 1000`}</pre>
</Concept>

export const deltaTimeCode = `// Compare delta-time motion vs fixed-increment motion

var x1 = 0;              // delta-time ball position
var x2 = 0;              // fixed-increment ball position
var speed = 150;          // 150 pixels per second
var lastTime = performance.now();

function loop(now) {
  // Calculate delta time in seconds
  var dt = (now - lastTime) / 1000;
  lastTime = now;

  ctx.clearRect(0, 0, 500, 300);

  // --- Ball 1: delta-time based (correct way) ---
  x1 += speed * dt;          // pixels/sec * seconds = pixels
  if (x1 > 550) x1 = -50;

  ctx.fillStyle = 'limegreen';
  ctx.beginPath();
  ctx.arc(x1, 100, 20, 0, Math.PI * 2);
  ctx.fill();
  ctx.fillStyle = 'gray';
  ctx.font = '13px monospace';
  ctx.fillText('speed * dt  (consistent on any monitor)', 60, 105);

  // --- Ball 2: fixed increment (wrong way) ---
  x2 += 2.5;                 // same speed only at 60fps!
  if (x2 > 550) x2 = -50;

  ctx.fillStyle = 'deeppink';
  ctx.beginPath();
  ctx.arc(x2, 200, 20, 0, Math.PI * 2);
  ctx.fill();
  ctx.fillStyle = 'gray';
  ctx.fillText('fixed += 2.5  (too fast at 120fps)', 60, 205);

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## The Game Loop Cycle

<p>Let's zoom out and see the full picture of what happens each frame. This is the heartbeat of every game, animation, and interactive canvas app.</p>

<Concept>
  <p><strong>The game loop: what happens 60 times per second:</strong></p>
  <pre>{`          ┌──────────────────────┐
          │  Browser calls your  │
          │  loop(timestamp)     │
          └──────────┬───────────┘
                     │
                     ▼
          ┌──────────────────────┐
          │  1. Calculate dt     │  dt = (now - last) / 1000
          │     (delta time)     │  last = now
          └──────────┬───────────┘
                     │
                     ▼
          ┌──────────────────────┐
          │  2. Clear canvas     │  ctx.clearRect(0, 0, w, h)
          └──────────┬───────────┘
                     │
                     ▼
          ┌──────────────────────┐
          │  3. Update state     │  position += velocity * dt
          │     (physics, AI,    │  check collisions
          │      input, logic)   │  handle user input
          └──────────┬───────────┘
                     │
                     ▼
          ┌──────────────────────┐
          │  4. Draw everything  │  ctx.fillRect, arc, etc.
          └──────────┬───────────┘
                     │
                     ▼
          ┌──────────────────────┐
          │  5. Request next     │  __raf = requestAnimation-
          │     frame            │    Frame(loop)
          └──────────┬───────────┘
                     │
                     └──── repeats forever ───┘`}</pre>
</Concept>

## Physics Basics: Position, Velocity, Acceleration

<p>Real-world objects do not teleport; they accelerate, cruise, and slow down. To make canvas animation feel physical and natural, we model three properties:</p>

<Concept>
  <p><strong>The three layers of motion:</strong></p>
  <pre>{`  Position     = where the object is right now (x, y)
  Velocity     = how fast it's moving and in what direction (vx, vy)
  Acceleration = how velocity is changing (ax, ay)

  Each frame:
    velocity += acceleration * dt
    position += velocity * dt

  Example: a ball falling due to gravity:
    ay = 500 (pixels/sec², pointing down)
    vy += 500 * dt    → velocity increases each frame
    y  += vy  * dt    → position changes by velocity

  This is Euler integration: simple and good enough for games.`}</pre>
</Concept>

export const physicsBasicsCode = `// Three balls showing different levels of physics

var ball1 = { x: 50, y: 175 };                           // position only
var ball2 = { x: 50, y: 175, vx: 100 };                  // position + velocity
var ball3 = { x: 50, y: 175, vx: 20, ax: 80 };           // position + velocity + acceleration

var lastTime = performance.now();
var elapsed = 0;

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;
  elapsed += dt;

  // Reset every 4 seconds
  if (elapsed > 4) {
    elapsed = 0;
    ball1.x = 50;
    ball2.x = 50; ball2.vx = 100;
    ball3.x = 50; ball3.vx = 20;
  }

  ctx.clearRect(0, 0, 500, 350);

  // Ball 1: Fixed position change (teleporting, no physics)
  ball1.x += 60 * dt;
  ctx.fillStyle = 'deeppink';
  ctx.beginPath();
  ctx.arc(ball1.x, 80, 18, 0, Math.PI * 2);
  ctx.fill();
  ctx.fillStyle = 'gray';
  ctx.font = '12px monospace';
  ctx.fillText('constant speed (no acceleration)', 60, 45);

  // Ball 2: Constant velocity
  ball2.x += ball2.vx * dt;
  ctx.fillStyle = 'cyan';
  ctx.beginPath();
  ctx.arc(ball2.x, 175, 18, 0, Math.PI * 2);
  ctx.fill();
  ctx.fillStyle = 'gray';
  ctx.fillText('constant velocity: vx = 100', 60, 140);

  // Ball 3: Acceleration: velocity changes over time!
  ball3.vx += ball3.ax * dt;      // acceleration changes velocity
  ball3.x += ball3.vx * dt;       // velocity changes position
  ctx.fillStyle = 'limegreen';
  ctx.beginPath();
  ctx.arc(ball3.x, 270, 18, 0, Math.PI * 2);
  ctx.fill();
  ctx.fillStyle = 'gray';
  ctx.fillText('acceleration: starts slow, gets faster', 60, 235);

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## Gravity Simulation: The Bouncing Ball

<p>Gravity is just acceleration pointing downward. Each frame, gravity adds to the ball's downward velocity, making it fall faster and faster. When the ball hits the floor, we reverse its vertical velocity (bounce) and multiply by a number less than 1 so it loses energy each bounce, just like a real ball. This demo also uses <code>ctx.shadowColor</code> and <code>ctx.shadowBlur</code> to add a glow effect around the ball; setting <code>shadowBlur</code> back to 0 disables it for subsequent draws.</p>

<Concept>
  <pre>{`  Gravity simulation each frame:
    vy += gravity * dt        // gravity pulls the ball down
    y  += vy * dt             // ball moves according to velocity

  When ball hits floor (y + radius > floorY):
    y  = floorY - radius      // snap to floor surface
    vy = vy * -0.7            // reverse direction, lose 30% energy
                               //  -0.7 = bounce + energy loss

  The ball bounces lower each time until it "settles" on the floor.`}</pre>
</Concept>

export const gravityBounceCode = `var ball = {
  x: 100, y: 50,
  vx: 120,              // horizontal speed (pixels/sec)
  vy: 0,                // vertical speed (starts at 0: ball is "dropped")
  radius: 18
};
var gravity = 600;       // pixels/sec² (pulls downward)
var bounceFactor = 0.7;  // how much energy is kept after bounce (0-1)
var floorY = 380;        // where the "ground" is
var lastTime = performance.now();

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;

  // --- Update physics ---
  ball.vy += gravity * dt;    // gravity accelerates the ball downward
  ball.x += ball.vx * dt;
  ball.y += ball.vy * dt;

  // Bounce off floor
  if (ball.y + ball.radius > floorY) {
    ball.y = floorY - ball.radius;     // snap to floor
    ball.vy = -ball.vy * bounceFactor; // reverse + lose energy
  }

  // Bounce off walls
  if (ball.x - ball.radius < 0) {
    ball.x = ball.radius;
    ball.vx = Math.abs(ball.vx);
  }
  if (ball.x + ball.radius > 500) {
    ball.x = 500 - ball.radius;
    ball.vx = -Math.abs(ball.vx);
  }

  // Bounce off ceiling
  if (ball.y - ball.radius < 0) {
    ball.y = ball.radius;
    ball.vy = Math.abs(ball.vy);
  }

  // --- Draw ---
  ctx.clearRect(0, 0, 500, 400);

  // Draw floor
  ctx.strokeStyle = '#1a1a2a';
  ctx.lineWidth = 2;
  ctx.beginPath();
  ctx.moveTo(0, floorY);
  ctx.lineTo(500, floorY);
  ctx.stroke();

  // Draw ball with shadow for depth
  ctx.fillStyle = 'cyan';
  ctx.shadowColor = 'cyan';
  ctx.shadowBlur = 12;
  ctx.beginPath();
  ctx.arc(ball.x, ball.y, ball.radius, 0, Math.PI * 2);
  ctx.fill();
  ctx.shadowBlur = 0;

  // Draw ball shadow on floor
  var shadowScale = 1 - (floorY - ball.y) / floorY;
  ctx.fillStyle = 'rgba(0, 229, 255, 0.15)';
  ctx.beginPath();
  ctx.ellipse(ball.x, floorY + 5, ball.radius * (0.5 + shadowScale * 0.5), 4, 0, 0, Math.PI * 2);
  ctx.fill();

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## Friction and Damping

<p>In real life, things slow down. A ball rolling on the ground loses speed to friction. A swinging pendulum gradually stops. In code, we simulate this by multiplying velocity by a number slightly less than 1 each frame. This is called <strong>damping</strong>.</p>

<Concept>
  <pre>{`  Damping (simple friction):
    velocity *= 0.99          // lose 1% speed each frame

  Problem: this is frame-rate dependent! At 120fps you
  apply 0.99 twice as often → things slow down faster.

  Frame-rate independent damping:
    velocity *= Math.pow(damping, dt)

  Example with damping = 0.01 (meaning "keep 1% per second"):
    At 60fps:  v *= pow(0.01, 0.0167) = v *= 0.926  each frame
    At 120fps: v *= pow(0.01, 0.0083) = v *= 0.962  each frame
    Both reach the same speed after 1 second!`}</pre>
</Concept>

export const frictionDampingCode = `// Click to launch a ball: it slows down from friction

var balls = [];
var damping = 0.02;  // fraction of speed remaining after 1 second
var lastTime = performance.now();

// Start with one ball
balls.push({
  x: 250, y: 175,
  vx: 300, vy: -200,
  hue: 200
});

addListener(canvas, 'click', function(e) {
  var rect = canvas.getBoundingClientRect();
  var mx = e.clientX - rect.left;
  var my = e.clientY - rect.top;
  // Launch from center toward click
  balls.push({
    x: 250, y: 175,
    vx: (mx - 250) * 3,
    vy: (my - 175) * 3,
    hue: Math.random() * 360
  });
  // Keep max 8 balls
  if (balls.length > 8) balls.shift();
});

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;

  ctx.clearRect(0, 0, 500, 350);

  ctx.fillStyle = 'gray';
  ctx.font = '12px monospace';
  ctx.fillText('click to launch balls: watch them slow down', 100, 20);

  for (var i = 0; i < balls.length; i++) {
    var b = balls[i];

    // Apply frame-rate independent damping
    var dampFactor = Math.pow(damping, dt);
    b.vx *= dampFactor;
    b.vy *= dampFactor;

    // Update position
    b.x += b.vx * dt;
    b.y += b.vy * dt;

    // Bounce off walls
    if (b.x < 10) { b.x = 10; b.vx = Math.abs(b.vx); }
    if (b.x > 490) { b.x = 490; b.vx = -Math.abs(b.vx); }
    if (b.y < 10) { b.y = 10; b.vy = Math.abs(b.vy); }
    if (b.y > 340) { b.y = 340; b.vy = -Math.abs(b.vy); }

    // Draw ball: opacity fades as it slows
    var speed = Math.sqrt(b.vx * b.vx + b.vy * b.vy);
    var alpha = Math.min(1, speed / 50 + 0.15);
    ctx.globalAlpha = alpha;
    ctx.fillStyle = 'hsl(' + b.hue + ', 70%, 60%)';
    ctx.beginPath();
    ctx.arc(b.x, b.y, 14, 0, Math.PI * 2);
    ctx.fill();
  }
  ctx.globalAlpha = 1;

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## Sine Wave Motion: Oscillation

<p><code>Math.sin()</code> is the animation secret weapon. It produces a smooth wave that oscillates between -1 and +1, repeating forever. Feed it a steadily increasing number (like time) and you get smooth back-and-forth motion for free, no physics needed.</p>

<Concept>
  <p><strong>Sine wave anatomy:</strong></p>
  <pre>{`  Math.sin(time) oscillates between -1 and +1:

   +1 ─  ╭──╮       ╭──╮       ╭──╮
         │  │       │  │       │  │
    0 ───┤  ├───────┤  ├───────┤  ├──── time →
         │  │       │  │       │  │
   -1 ─  ╰──╯       ╰──╯       ╰──╯

  To control oscillation:
    Amplitude:  Math.sin(t) * 50       → swings 50px each way
    Frequency:  Math.sin(t * 3)        → 3x faster oscillation
    Offset:     200 + Math.sin(t) * 50 → centers at y=200

  Use cases: bobbing, pulsing, floating, breathing effects`}</pre>
</Concept>

export const sineWaveCode = `// Three uses of sine: bobbing, pulsing, and orbiting

var lastTime = performance.now();
var time = 0;

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;
  time += dt;

  ctx.clearRect(0, 0, 500, 400);
  ctx.font = '12px monospace';
  ctx.fillStyle = 'gray';

  // --- 1. Bobbing (vertical sine) ---
  ctx.fillText('bobbing: y = 70 + sin(time * 2) * 30', 160, 25);
  var bobY = 70 + Math.sin(time * 2) * 30;
  ctx.fillStyle = 'cyan';
  ctx.beginPath();
  ctx.arc(100, bobY, 18, 0, Math.PI * 2);
  ctx.fill();

  // --- 2. Pulsing (size sine) ---
  ctx.fillStyle = 'gray';
  ctx.fillText('pulsing: r = 20 + sin(time * 3) * 10', 160, 170);
  var pulseR = 20 + Math.sin(time * 3) * 10;
  ctx.fillStyle = 'deeppink';
  ctx.beginPath();
  ctx.arc(100, 175, pulseR, 0, Math.PI * 2);
  ctx.fill();

  // --- 3. Orbiting (sin + cos = circle) ---
  ctx.fillStyle = 'gray';
  ctx.fillText('orbit: x = cos(t)*60, y = sin(t)*60', 160, 310);
  var orbitX = 100 + Math.cos(time * 1.5) * 60;
  var orbitY = 320 + Math.sin(time * 1.5) * 60;

  // Draw orbit path
  ctx.strokeStyle = '#1a1a2a';
  ctx.lineWidth = 1;
  ctx.beginPath();
  ctx.arc(100, 320, 60, 0, Math.PI * 2);
  ctx.stroke();

  // Draw center dot
  ctx.fillStyle = '#1a1a2a';
  ctx.beginPath();
  ctx.arc(100, 320, 4, 0, Math.PI * 2);
  ctx.fill();

  // Draw orbiting dot
  ctx.fillStyle = 'limegreen';
  ctx.beginPath();
  ctx.arc(orbitX, orbitY, 14, 0, Math.PI * 2);
  ctx.fill();

  // --- Draw a sine wave visualization on the right ---
  ctx.strokeStyle = 'cyan';
  ctx.lineWidth = 2;
  ctx.beginPath();
  for (var px = 0; px < 200; px++) {
    var waveX = 270 + px;
    var waveY = 200 + Math.sin((px / 30) + time * 2) * 80;
    if (px === 0) ctx.moveTo(waveX, waveY);
    else ctx.lineTo(waveX, waveY);
  }
  ctx.stroke();

  ctx.fillStyle = 'gray';
  ctx.fillText('the sine wave itself', 310, 105);

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## Lerp: The Most Useful Animation Function

<p><strong>Lerp</strong> stands for <em>linear interpolation</em>. It answers the question: "Given a start value and an end value, what is the value at some percentage between them?" It is a single line of code and it is absurdly useful.</p>

<Concept>
  <p><strong>Lerp explained:</strong></p>
  <pre>{`  function lerp(start, end, t) {
    return start + (end - start) * t;
  }

  t = 0   → returns start
  t = 0.5 → returns halfway between start and end
  t = 1   → returns end

  Example: lerp(100, 300, 0.25) = 100 + (300-100) * 0.25 = 150

  The "smooth follow" trick:
    x = lerp(x, targetX, 0.1)   // each frame, move 10% closer

  This creates smooth, decelerating motion: the object
  moves fast when far away and slows as it approaches.
  It's how most UI animations work!`}</pre>
</Concept>

export const lerpDemoCode = `// Move your mouse: the circle smoothly follows using lerp

function lerp(a, b, t) {
  return a + (b - a) * t;
}

var circleX = 250;

## Particle Trails Using Semi-Transparent Clear

<p>Normally we clear the canvas completely each frame with <code>clearRect</code>. But there is a beautiful trick: instead of clearing fully, draw a semi-transparent rectangle over the entire canvas. The previous frame fades but does not disappear instantly, creating a trail or "ghosting" effect.</p>

<Concept>
  <pre>{`  Normal clear (no trails):
    ctx.clearRect(0, 0, w, h);     // 100% erase, clean slate

  Trail effect (semi-transparent overlay):
    ctx.fillStyle = 'rgba(10, 10, 15, 0.1)';  // dark + 10% opaque
    ctx.fillRect(0, 0, w, h);

  Lower alpha = longer trails (more ghosting)
  Higher alpha = shorter trails (fades faster)

  rgba(10, 10, 15, 0.05) → long comet tails
  rgba(10, 10, 15, 0.15) → medium trails
  rgba(10, 10, 15, 0.4)  → short fade`}</pre>
</Concept>

export const particleTrailsCode = `// Particles orbit a center point, leaving colorful trails

var particles = [];
var centerX = 250, centerY = 200;
var time = 0;
var lastTime = performance.now();

// Create particles at different orbit distances and speeds
for (var i = 0; i < 12; i++) {
  particles.push({
    angle: (Math.PI * 2 / 12) * i,
    radius: 50 + Math.random() * 120,
    speed: 0.5 + Math.random() * 1.5,
    size: 3 + Math.random() * 5,
    hue: (i / 12) * 360
  });
}

// Fill background once at start
ctx.fillStyle = 'black';
ctx.fillRect(0, 0, 500, 400);

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;
  time += dt;

  // Semi-transparent overlay instead of full clear: creates trails!
  ctx.fillStyle = 'rgba(10, 10, 15, 0.08)';
  ctx.fillRect(0, 0, 500, 400);

  for (var i = 0; i < particles.length; i++) {
    var p = particles[i];
    p.angle += p.speed * dt;

    // Calculate position on orbit
    var x = centerX + Math.cos(p.angle) * p.radius;

    ctx.fill();
  }
  ctx.shadowBlur = 0;

}
__raf = requestAnimationFrame(loop);`;

## FPS Counter: Measuring Performance

<p>How do you know if your animation is running smoothly? Build an FPS counter. Count how many frames happen each second. If the number drops below 60, your drawing code is too slow and you need to optimize.</p>

<Concept>
  <pre>{`  FPS counter strategy:
    1. Count frames in a running window
    2. Every 0.5 seconds, calculate: frames / elapsed = fps
    3. Display the number

  Alternative (simpler): fps = 1 / deltaTime
    But this fluctuates wildly frame-to-frame.
    Averaging over 0.5-1 second looks much better.`}</pre>
</Concept>

export const fpsCounterCode = `// An FPS counter overlaid on an animated scene

var fps = 60;
var frameCount = 0;
var fpsTimer = 0;
var lastTime = performance.now();
var time = 0;

// Create some objects to animate
var dots = [];
for (var i = 0; i < 80; i++) {
  dots.push({
    angle: Math.random() * Math.PI * 2,
    dist: 30 + Math.random() * 130,
    speed: 0.3 + Math.random() * 2,
    size: 2 + Math.random() * 4,
    hue: Math.random() * 360
  });
}

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;
  time += dt;

  // --- FPS calculation ---
  frameCount++;
  fpsTimer += dt;
  if (fpsTimer >= 0.5) {              // update display every 0.5s

    frameCount = 0;
    fpsTimer = 0;

  // --- Draw scene ---
  ctx.clearRect(0, 0, 500, 350);

  var cx = 250, cy = 195;

    ctx.arc(x, y, d.size, 0, Math.PI * 2);
    ctx.fill();

  ctx.globalAlpha = 1;

## Easing Functions: Natural-Feeling Motion

<p>Linear motion, moving the same amount every frame, looks robotic. Real objects ease in (start slow), ease out (slow down at the end), or both. Easing functions take a progress value <code>t</code> (from 0 to 1) and return a curved version of it.</p>

<Concept>
  <p><strong>Common easing functions:</strong></p>
  <pre>{`  linear(t)       = t              flat line, constant speed
  easeInQuad(t)    = t * t          starts slow, ends fast
  easeOutQuad(t)   = t * (2 - t)   starts fast, ends slow
  easeInOutCubic   = smooth S-curve slow → fast → slow

  bounce(t)  = ball-drop effect
  elastic(t) = springy overshoot

  All take t from 0→1 and return a modified 0→1 value.

  Usage:
    var t = elapsed / duration;        // 0 → 1 over time
    var eased = easeInOutCubic(t);     // apply curve
    var x = startX + (endX - startX) * eased;  // interpolate`}</pre>
</Concept>

export const easingComparisonCode = `// Six easing functions compared side by side

function linear(t) { return t; }
function easeInQuad(t) { return t * t; }
function easeOutQuad(t) { return t * (2 - t); }
function easeInOutCubic(t) {
  return t < 0.5 ? 4*t*t*t : 1 - Math.pow(-2*t+2, 3) / 2;
}

  var n1 = 7.5625, d1 = 2.75;
  if (t < 1/d1) return n1*t*t;
  if (t < 2/d1) return n1*(t-=1.5/d1)*t+0.75;
  if (t < 2.5/d1) return n1*(t-=2.25/d1)*t+0.9375;

## Springs and Elastic Motion

<p>A spring simulation gives you organic, bouncy motion that is hard to achieve with easing curves alone. The idea comes from Hooke's law: the further a spring is stretched from its rest position, the harder it pulls back. Add some damping so it eventually settles.</p>

<Concept>
  <p><strong>Spring physics:</strong></p>
  <pre>{`  Spring force = -stiffness * displacement
  Damping force = -damping * velocity

  Each frame:
    var displacement = position - target;
    var springForce  = -stiffness * displacement;
    var dampingForce = -damping * velocity;
    velocity += (springForce + dampingForce) * dt;
    position += velocity * dt;

  stiffness controls how "snappy" it is  (higher = faster)
  damping controls how quickly it settles (higher = less bounce)

  Low stiffness + low damping  = slow, bouncy jelly
  High stiffness + low damping = fast, ringy vibration
  High stiffness + high damping = fast snap, minimal bounce`}</pre>
</Concept>

export const springMotionCode = `// Click anywhere: the ball springs to that position

var ball = { x: 250, y: 175, vx: 0, vy: 0 };
var target = { x: 250, y: 175 };
var stiffness = 180;   // how hard the spring pulls
var damping = 8;        // how quickly oscillation dies
var lastTime = performance.now();

addListener(canvas, 'click', function(e) {
  var rect = canvas.getBoundingClientRect();
  target.x = e.clientX - rect.left;
  target.y = e.clientY - rect.top;
});

function loop(now) {
  var dt = (now - lastTime) / 1000;
  lastTime = now;
  // Cap dt to avoid explosion on tab-switch
  if (dt > 0.1) dt = 0.016;

  // --- Spring physics ---
  var dx = ball.x - target.x;
  var dy = ball.y - target.y;

  // Spring force pulls toward target
  var fx = -stiffness * dx;
  var fy = -stiffness * dy;

  // Damping force resists velocity
  fx -= damping * ball.vx;
  fy -= damping * ball.vy;

  // Apply forces
  ball.vx += fx * dt;
  ball.vy += fy * dt;
  ball.x += ball.vx * dt;
  ball.y += ball.vy * dt;

  // --- Draw ---
  ctx.clearRect(0, 0, 500, 350);

  // Target crosshair
  ctx.strokeStyle = '#1a1a2a';
  ctx.lineWidth = 1;
  ctx.beginPath();
  ctx.moveTo(target.x - 20, target.y);
  ctx.lineTo(target.x + 20, target.y);
  ctx.moveTo(target.x, target.y - 20);
  ctx.lineTo(target.x, target.y + 20);
  ctx.stroke();

  // Spring line
  ctx.strokeStyle = 'limegreen';
  ctx.lineWidth = 2;
  ctx.setLineDash([4, 4]);
  ctx.beginPath();
  ctx.moveTo(ball.x, ball.y);
  ctx.lineTo(target.x, target.y);
  ctx.stroke();
  ctx.setLineDash([]);

  // Ball
  ctx.fillStyle = 'limegreen';
  ctx.shadowColor = 'limegreen';
  ctx.shadowBlur = 15;
  ctx.beginPath();
  ctx.arc(ball.x, ball.y, 18, 0, Math.PI * 2);
  ctx.fill();
  ctx.shadowBlur = 0;

  // Label
  ctx.fillStyle = 'gray';
  ctx.font = '12px monospace';
  ctx.fillText('click to set target: ball springs to it', 115, 20);

  __raf = requestAnimationFrame(loop);
}
__raf = requestAnimationFrame(loop);`;

## Practical: Analog Clock

<p>A clock combines several animation concepts: trigonometry for hand positions, real time data from <code>Date</code>, and the game loop for smooth second-hand ticking. This demo reads the actual current time and displays it.</p>

  ctx.rotate(angle);

    ctx.beginPath();
    ctx.moveTo(cx + Math.cos(mAngle) * (radius - 6), cy + Math.sin(mAngle) * (radius - 6));
    ctx.lineTo(cx + Math.cos(mAngle) * (radius - 3), cy + Math.sin(mAngle) * (radius - 3));
    ctx.stroke();
  }

  // --- Draw hands ---
  // Hour hand
  var hourAngle = ((hours + minutes / 60) / 12) * Math.PI * 2 - Math.PI / 2;
  drawHand(hourAngle, 60, 5, 'lightgray');

  // Minute hand
  var minAngle = ((minutes + smoothSec / 60) / 60) * Math.PI * 2 - Math.PI / 2;
  drawHand(minAngle, 85, 3, 'gray');

  // Second hand (smooth sweep)
  var secAngle = (smoothSec / 60) * Math.PI * 2 - Math.PI / 2;
  drawHand(secAngle, 95, 1.5, 'deeppink');

  // Center cap
  ctx.fillStyle = 'deeppink';
  ctx.beginPath();
  ctx.arc(cx, cy, 5, 0, Math.PI * 2);
  ctx.fill();

  // Digital time below
  ctx.fillStyle = 'dimgray';
  ctx.font = '12px monospace';
  var timeStr = (hours === 0 ? 12 : hours) + ':' +
    (minutes < 10 ? '0' : '') + minutes + ':' +
    (seconds < 10 ? '0' : '') + seconds;
  ctx.textAlign = 'center';
  ctx.fillText(timeStr, cx, cy + 45);
  ctx.textAlign = 'start';

  __raf = requestAnimationFrame(loop);
}
loop();`;

## What You Learned

<Concept>
  <p><strong>Lesson 6 recap, everything you need for canvas animation:</strong></p>
  <pre>{`  Core loop:      clear → update → draw → requestAnimationFrame
  Delta time:      distance = speed * dt  (frame-rate independent)
  Physics:         velocity += acceleration * dt; position += velocity * dt
  Gravity:         acceleration pointing down + bounce on collision
  Friction:        velocity *= Math.pow(damping, dt)
  Sine waves:      Math.sin(time * freq) * amplitude + offset
  Lerp:            start + (end - start) * t  (smooth follow)
  Springs:         force = -stiffness * displacement - damping * velocity
  Trails:          semi-transparent fillRect instead of clearRect
  Color cycling:   hsl(hue, sat, light) where hue increases over time
  Easing:          curve functions that map t(0→1) to modified t(0→1)
  FPS counter:     count frames per half-second interval
  __raf:           always store requestAnimationFrame ID for cleanup`}</pre>
</Concept>
